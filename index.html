Android软件栈

Android系统是由不同层次构成的软件栈，每层都有彼此独立的功能并向上层提供特定的服务。
由下到上依次是：

  • 第一层：linux内核层，主要实现驱动模块。显示驱动，相机驱动，蓝牙驱动，WIFI驱动，声卡驱动，电源管理等等。属于一个移动操作系统的基本功能单元。
  • 第二层：本地库，运行时环境（虚拟机)。数据库：sqlite，libc，界面管理，多媒体框架，webkit等。Android系统调度的核心
  • 第三层：应用程序框架：实现Android应用程序和本地库以及内核的交互。acitivity manager ，数据库索引 view system 位置管理，电话管理，包管理，资源管理
  • 第四层：应用程序层，通过第三层提供的API来实现应用软件的功能。直接封装好的应用程序（已经是成品了）

    Android框架层分析

    提供系统服务和和硬件交互的接口
    程序启动—>载入框架层组件管理周期—>event handler启动会执行一些具体的代码（入点）
    例如：位置这个event是从GPS传感器来的，SMS是从蜂窝芯片得来的。（即数据是从底层被回调上来）
    有一些入点会在框架层注册（对应在Manifest里面声明）—>被相关event触发
    Android应用的功能是由一个完整的事件处理程序（Event Handler）组成。Android程序的控制流图和调用关系均是由Event Handler之间的处理顺序和耦合调用组成，有时候不同的Event Handler之间是异步的，多线程的，还有可能通过Intent来通信。在生成调用图的时候，这些问题一定要考虑到。

以上是一些研究过程中的理解，具体细节的展开，等以后在细细讲解。

发表于 2018-03-10

局限性：伪并发，若命令之间相互交互，不建议使用shell脚本

循环体内使用{}&

提示符：[~/bin/multiple_process$]
循环体内的命令全部进入后台，没有占用脚本主线程的时间
问题：循环体内的命令全部进入后台，相当于开了和循环次数相同的进程数，即开了和循环次数相等的终端。

{}&,done后面加wait

等待当前脚本进程下的子进程结束，再运行后面的语句。
问题：一旦循环次数过多，操作系统会瞬间创建所有的紫禁城，会非常消耗资源，如果循环体内的命令很消耗资源，则会崩溃。

2.${var##*.}
该命令的作用是去掉变量var从左边算起的最后一个’.’字符及其左边的内容，返回从左边算起的最后一个’.’（不含该字符）的右边的内容。使用例子及结果如下：

如果文件的后缀不仅有一个，例如，file.tar.gz命令，${var##.}仅能提取最后一个后缀，而我想提取tar.gz时该怎么办？那么就要用下面所说的${var#.}命令了。

3.${var#*.}
该命令的作用是去掉变量var从左边算起的第一个’.’字符及其左边的内容，返回从左边算起第一个.（不含该字符）的右边部分的内容。使用例子及结果如下：

当我们需要建立一个与文件名相同名字（没有后缀）的目录与对应的文件相对应时，就可以使用该命令来进行操作。例如，解压文件的情况就与此类似，我们压缩文件file.zip时，会在与file.zip同级目录下建立一个名为file的目录。

${}总结：
其实${}并不是专门为提取文件名或目录名的，它的使用是变量的提取和替换等等操作，它可以提取非常多的内容，并不一定是上面五个例子中的’/‘或’.’。也就是说，上面的使用方法只是它使用的一个特例。

使用basename和dirname

${}并不是专门为提取文件名和目录名设计的命令，那么basename和dirname命令就是专门为做这一件事而已准备的了。
1.basename:
该命令的作用是从路径中提取出文件名，使用方法为basename NAME [SUFFIX]。
2.dirname:
该命令的作用是从路径中提取出目录名，使用方法为 dirname NAME
